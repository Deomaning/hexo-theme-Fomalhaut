# AI提示词工程实战

## 核心原则

### 1. 清晰具体
❌ 模糊: "写个代码"
✅ 清晰: "用Python写一个函数，接收一个整数列表，返回其中的最大值和最小值"

### 2. 提供上下文
```
背景: 我们是一个电商平台
任务: 设计一个订单状态机
约束: 
- 状态: 待支付、已支付、已发货、已签收、已取消
- 只能从未支付到已支付，不能逆向
```

### 3. 指定输出格式
```
请以JSON格式输出，包含以下字段:
- status: 状态码
- message: 状态描述
- data: 具体数据
```

## 高级技巧

### Chain-of-Thought
让AI分步骤思考：
```
请逐步分析这个问题：
1. 首先理解问题要求
2. 然后列出可能的解决方案
3. 评估每个方案的优缺点
4. 给出最终建议
```

### Few-Shot示例
```
任务: 将中文翻译成英文

示例1:
输入: 你好
输出: Hello

示例2:
输入: 谢谢
输出: Thank you

现在请翻译:
输入: 很高兴见到你
输出: 
```

### 角色设定
```
你是一位经验丰富的Java架构师，擅长设计高并发系统。
请帮我设计一个秒杀系统的核心架构。
```

## 常见陷阱

1. **提示词注入**: 用户输入包含恶意指令
2. **上下文过长**: 超出模型上下文限制
3. **歧义理解**: AI理解偏差导致错误输出
4. **过度依赖**: 不验证AI输出直接用于生产
